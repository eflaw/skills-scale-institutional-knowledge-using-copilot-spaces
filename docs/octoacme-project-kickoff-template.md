# OctoAcme — Project Kickoff Meeting Template

This template guides project managers and team leads in conducting effective project kickoff meetings that align all stakeholders and set the team up for success.

---

## Pre-Kickoff Preparation (1 week before)

### Materials to Prepare
- [ ] Project Charter/One-pager completed and reviewed
- [ ] Stakeholder list and communication plan document
- [ ] Initial risk register
- [ ] Draft project timeline and milestones
- [ ] Resource allocation and team roster
- [ ] Initial backlog outline

### Pre-Kickoff Communications
- [ ] Send meeting invite with clear objective and agenda
- [ ] Attach project charter and supporting documents
- [ ] Request that participants review materials before meeting
- [ ] Confirm attendance from all key roles

---

## Kickoff Meeting Agenda (90-120 minutes)

### 1. Welcome & Objectives (5 min)
**Facilitator**: Project Manager
- Welcome and thank you for participation
- Brief overview of meeting objectives
- Agenda review and time expectations

### 2. Project Vision & Charter (15 min)
**Presenter**: Product Manager
- Problem statement and business context
- Project objectives and success metrics
- Key stakeholders and dependencies
- High-level timeline and major milestones
- Q&A

### 3. Team Roles & Responsibilities (15 min)
**Presenter**: Project Manager
- Introduce team members by role
- Review core responsibilities from [Roles & Personas](octoacme-roles-and-personas.md)
- Clarify decision-making authority and escalation paths
- Communication expectations and cadence
- Q&A

### 4. Scope & Initial Backlog (20 min)
**Presenter**: Product Manager
- Product features and user stories (high-level)
- Scope boundaries and what's out of scope
- Initial backlog prioritization
- Acceptance criteria and Definition of Done
- Q&A

### 5. Project Plan & Timeline (15 min)
**Presenter**: Project Manager
- Release phases and milestones
- Key dates and dependencies
- Resource allocation and capacity
- Risk management approach
- Q&A

### 6. Communication & Collaboration (10 min)
**Presenter**: Project Manager
- Communication plan and cadence
  - Daily standups (time and format)
  - Weekly syncs (PM + PdM)
  - Bi-weekly delivery demos
  - Monthly stakeholder updates
- Escalation paths and contact information
- Documentation and decision-making processes
- Q&A

### 7. Team Norms & Working Agreements (10 min)
**Facilitator**: Project Manager (Collaborative Discussion)
- Working hours and timezone considerations
- Definition of "ready" for work
- Code review and testing expectations
- Meeting attendance expectations
- Respectful communication and psychological safety
- How decisions will be made
- Q&A and confirmation of agreement

### 8. Risks & Mitigation (10 min)
**Presenter**: Project Manager
- Initial risk register overview
- High-impact risks and mitigation plans
- Risk monitoring and escalation
- Team input on anticipated risks
- Q&A

### 9. Next Steps & Action Items (5 min)
**Facilitator**: Project Manager
- Review action items captured
- Confirm owners and due dates
- Schedule first standups and syncs
- Confirm artifact storage location (repo, wiki, etc.)

### 10. Closing Remarks (5 min)
**Presenter**: Project Manager or Project Sponsor
- Reinforce project importance
- Confidence in team capability
- Commit to removing blockers
- Thank you for participation

## Post-Kickoff Activities (within 1 week)

### Project Manager Actions
- [ ] Document meeting notes and decisions
- [ ] Send recap email with key takeaways
- [ ] Create initial sprint/iteration backlog
- [ ] Set up project board (GitHub Projects, Jira, etc.)
- [ ] Schedule recurring meetings (standups, syncs, demos)
- [ ] Create shared documentation space
- [ ] Establish communication channels (Slack, email, etc.)

### All Team Members
- [ ] Complete any pre-sprint training or onboarding
- [ ] Review project charter and initial backlog
- [ ] Set up development environments
- [ ] Confirm tool access (GitHub, CI/CD, monitoring, etc.)
- [ ] Review team working agreements

### Product Manager
- [ ] Finalize detailed backlog items with acceptance criteria
- [ ] Prepare first sprint planning meeting
- [ ] Conduct stakeholder alignment meetings if needed

### QA Lead
- [ ] Review acceptance criteria and test approach
- [ ] Prepare test plan template and environment requirements
- [ ] Coordinate with DevOps on test environment setup

### UX Designer (if applicable)
- [ ] Begin user research or design activities
- [ ] Schedule design review sessions with stakeholders
- [ ] Prepare design workflow and deliverables timeline

### DevOps Engineer
- [ ] Set up development, staging, and test environments
- [ ] Configure CI/CD pipeline
- [ ] Prepare deployment checklist and runbook

## Kickoff Meeting Checklist

### Before Meeting
- [ ] Charter and supporting docs prepared and shared
- [ ] Meeting room/Zoom link reserved
- [ ] All participants confirmed
- [ ] Agenda and materials sent in advance
- [ ] Visual aids or slides prepared

### During Meeting
- [ ] Start on time
- [ ] All participants can hear/see clearly
- [ ] Facilitator captures action items and decisions
- [ ] Encourage questions and participation
- [ ] Stay on time for each agenda item
- [ ] Confirm next steps before closing

### After Meeting
- [ ] Notes documented and shared within 1 day
- [ ] Action items captured with owners and dates
- [ ] Decisions documented in decision log
- [ ] Follow-up tasks assigned and tracked
- [ ] Next meetings scheduled
- [ ] Project artifacts organized and accessible

## Tips for Effective Kickoff

**Set a Positive Tone**
- Express confidence in the team and project success
- Create psychological safety for questions and concerns
- Emphasize collaboration and shared accountability

**Maximize Engagement**
- Encourage participation and questions throughout
- Use interactive elements (polling, group discussion, breakouts)
- Give team members opportunity to share concerns early

**Be Clear on Expectations**
- Explicitly state communication norms and escalation paths
- Clarify decision-making authority
- Set realistic timelines and manage expectations

**Document Everything**
- Capture decisions and action items in real-time
- Share notes within 24 hours
- Create decision log for future reference

**Follow Up Promptly**
- Assign action item owners and dates immediately
- Track follow-up items to completion
- Report progress in first standup/sync
